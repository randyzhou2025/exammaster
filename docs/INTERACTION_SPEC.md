# 人工智能训练师备考 H5 — 交互说明（原型对齐版）

**文档性质**：与可运行原型（`/Users/Randy/Cursor/ExamMaster`）同步的交互规格，供评审与研发对齐。  
**版本**：0.2  
**约束重申**：背题模式不参与首次统计；多选须与标准答案选项集合完全一致才得分。

---

## 1. 全局交互原则

| 原则 | 说明 |
|------|------|
| 触控目标 | 主要按钮与选项行高 ≥ 44px（CSS 约 `py-3` 及以上）。 |
| 不可逆操作 | 「清除全部数据」「退出考试」「交卷」使用系统 `confirm` 二次确认（原型）；正式版可换自研弹窗。 |
| 状态可见 | 顶栏分段控件明确当前为「答题 / 背题」；模考顶栏常驻倒计时。 |
| 反馈时效 | 单选/判断在答题模式下点选项即判分；多选在答题模式下需点「确认答案」后判分。 |

---

## 2. 信息架构与路由

| 路径 | 页面 | 说明 |
|------|------|------|
| `/` | 考试中心（首页） | 顺序练习、模拟考试、错题本、收藏入口。 |
| `/sequential` | 顺序练习仪表盘 | 统计 + 继续练习 + 随机/未做/错题快捷。 |
| `/practice/session` | 做题页 | 依赖 store 中 `practice` 上下文；无上下文时引导回首页。 |
| `/practice/wrong` | 错题入口 | 初始化 `kind=wrong` 后 `replace` 至 `/practice/session`。 |
| `/practice/favorite` | 收藏入口 | 同上，`kind=favorite`。 |
| `/mock` | 模考组卷说明 | 展示规则；组卷失败仅提示，不进入考试。 |
| `/mock/session` | 考试中 | 答题卡、上下题、交卷、倒计时归零自动交卷。 |
| `/mock/result` | 成绩单 | 依赖导航 `state`；直接打开 URL 无 state 时展示空态。 |
| `/settings` | 设置 | 清除数据、规则说明。 |

---

## 3. 页面级交互

### 3.1 首页 `/`

- **顺序练习**：进入 `/sequential`。  
- **模拟考试**：进入 `/mock`。  
- **错题本 / 收藏**：进入 `/practice/wrong` 或 `/practice/favorite`。  
- **设置**：`/settings`。

### 3.2 顺序练习仪表盘 `/sequential`

- **返回**：浏览器历史返回。  
- **继续练习**：写入 `practice = { kind: sequential, index: 0, ... }`，跳转 `/practice/session`。  
- **随机 / 未做 / 错题**：同上，`kind` 分别为 `random` / `unanswered` / `wrong`。  
- **统计**：未做、已做、错题本在册数、首次答题正确率（见 §4）。

### 3.3 做题页 `/practice/session`

**顶栏**

- 返回：结束练习（`exitPractice`）并 `navigate(-1)`。  
- 分段控件：**答题模式** / **背题模式**；切换时重置当前题本地作答态（不自动提交）。  
- 设置：`/settings`。  
- 辅助文案：明确背题模式不写首次统计。

**题型与选项**

- 题型标签：判断 / 单选 / 多选。  
- **答题模式**  
  - 判断 / 单选：单击选项立即判分、展示解析与「下一题」。  
  - 多选：多选后点「确认答案」判分；判分规则为全对才得分（与模考一致）。  
- **背题模式**  
  - 进入题目即展示正确项高亮与解析区；选项不可交互（点击不生效）。  
  - 「下一题」用于推进；**不调用** `submitPracticeAnswer`。

**解析区**

- 判分后（答题模式）或进入题目后（背题模式）展示「试题详解」与「本题技巧」（若有数据）。

**底栏**

- 收藏：切换当前题收藏状态（与模式无关）。  
- 首对 / 首错：聚合自全库首次答题结果（见 §4）。  
- 题单：打开底部抽屉网格；选题号 `setPracticeIndex` 并关闭抽屉，同时重置当前题本地态。

**空集合**

- 若 `orderedIds.length === 0`（如错题本为空），原型仍进入做题页会显示空态；**建议后续**：入口侧拦截并 Toast「暂无题目」。

### 3.4 模考 `/mock` → `/mock/session` → `/mock/result`

**组卷页**

- 「开始考试」：调用组卷算法；若题库不足以凑齐官方配比（40/140/10），展示错误文案，**不进入**考试。  
- 演示题库题量不足时必失败，为预期行为。

**考试中**

- 倒计时：每秒刷新；归零触发自动交卷（等同超时交卷）。  
- 退出：确认后 `abortMockExam`，回到 `/mock`。  
- 交卷：确认后计算得分（多选全对才得分），写入历史记录，导航至成绩单并携带 `state`。  
- 答题卡：展示已答/未答态；选题跳转当前题。

**成绩单**

- 展示得分、满分、是否合格、错题数。  
- 「再考一次」→ `/mock`；「返回首页」→ `/`。

## 4. 数据与统计口径（交互后果）

| 行为 | 是否写入 `firstAnswerMode` / 错题本 |
|------|-------------------------------------|
| 答题模式下首次判分 | 写入 `firstAnswerMode`；首错进入错题本（`remediated=false`）；首对标记 `remediated=true`。 |
| 答题模式下后续判分 | 不改变 `firstAnswerMode`；若曾首错，任一次答对则 `remediated=true`，移出错题本。 |
| 背题模式任意操作 | **不写入**上述字段（不参与首次统计与错题本）。 |

**底栏「首对 / 首错」**：对题库中每题，仅统计 `firstAnswerMode` 为 `correct` / `wrong` 的数量。
